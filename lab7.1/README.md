'# Звіт до лабораторної роботи №6  
## Тема: AI Агенти з Google ADK  
### Мета роботи  
Навчитись створювати AI агентів з використанням Google ADK (Python) та Poetry для управління залежностями проєкту.

---

## Виконання роботи

### 1. Підготовка робочого середовища
Для виконання лабораторної роботи було підготовлено середовище розробки та встановлено необхідні інструменти.

- **Версія Python:** `3.14.3`  
- **Версія Poetry:** `2.3.4`

Файл **`poetry.lock`** використовується для фіксації точних версій усіх залежностей (бібліотек), включаючи їхні власні залежності, що гарантує відтворюваність середовища.

**Основні команди Google ADK:**
- `create` — створення структури агента  
- `run` — запуск агента в інтерактивному режимі  
- `web` — запуск локального веб-інтерфейсу  

---

### 2. Створення першого агента (Time Agent)
Було створено першого AI агента, який повідомляє поточний час у заданому місті.

- **Клас `Agent`** — основний клас ADK, який поєднує мовну модель, системні інструкції та інструменти.
- **Параметр `tools`** — список Python-функцій, які агент може викликати автоматично.
- **Функція `get_current_time`** — приймає назву міста та повертає поточний системний час у форматі `HH:MM:SS`.

### 4. Агент-помічник для студентів
Було створено агента `student_helper`, який виконує роль навчального асистента.

Агент:
- пояснює теоретичні поняття з програмування;
- надає приклади коду;
- відповідає у зрозумілій та структурованій формі.

### 5. Креативний агент та робота з конфігурацією
Було створено агента `creative_writer` для генерації художніх текстів.

Під час спроби використання об’єкта `GenerateContentConfig` з параметрами `temperature`, `top_k`, `top_p` виникла помилка валідації **Pydantic (`extra_forbidden`)**, оскільки поточна версія `LlmAgent` не підтримує параметр `config`.

Через це агент був протестований без додаткових параметрів конфігурації та успішно згенерував казку про дружбу між людиною та роботом.
